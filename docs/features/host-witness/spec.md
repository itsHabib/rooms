# Host witness — unforgeable egress evidence per room

**Status:** v0 spec — network witness only. Filesystem witness is deferred (see
"Explicitly out of scope").

## Problem

Every evidence artifact a room emits today is reported from *inside* the guest:
`changeset.json` is enumerated over SSH by a root-capable identity in the guest,
and `result.patch` is generated by guest-side git. A careless agent is
observable; an adversarial or compromised guest can forge or suppress its own
report. Rooms' evidence is therefore trustworthy only against honest workloads
— which caps what the portfolio's verification and capability planes can build
on it (see `docs/product-directions.md`, "Software Customs").

One surface crosses the VM boundary through an object the guest cannot reach:
**the network tap**. Every packet the guest sends physically transits
`tap-fc<k>` on the host. Evidence recorded there cannot be forged or hidden
from within the room.

## Goal

Each room optionally records its own egress on the host side and emits a
`witness.json` artifact into the run's `out/` directory: the set of network
destinations the guest actually contacted, observed from outside the trust
boundary.

## Functional requirements

1. **Opt-in flag.** `rooms run --witness` enables capture. Off by default (no
   new overhead or failure modes on existing paths).
2. **Capture scope.** Capture starts before the guest can transmit (at or
   before VMM start, after slot allocation binds the tap name) and stops at
   teardown. The capture is bound to the room's *own* tap (`tap-fc<k>` from its
   allocated slot) — never another slot's.
3. **Raw capture artifact.** Packets are written to `witness.pcap` in the room
   work dir. Implementation may shell out to `tcpdump` (present on the
   rooms-host) with a capped file size; a missing `tcpdump` with `--witness`
   requested is a hard, early error (fail closed — do not silently run
   unwitnessed).
4. **Summarized artifact.** After teardown, the host derives `witness.json`
   (schema below) from the pcap and writes it **atomically** into `out/`
   alongside `result.json`/`changeset.json`. The pcap itself is copied into
   `out/` too (subject to the size cap) so the raw evidence survives the room.
5. **Schema** (`schema_version: 1`):
   - `tap`: the captured interface name.
   - `capture_complete`: bool — false if capture started late, died early, or
     hit the size cap (truncation must be *visible*, never silent).
   - `destinations`: deduplicated list of `{ip, port, proto, packets}` for
     egress flows, **keyed on the destination, never on a trusted source**.
     A root-compromised guest can forge its IPv4 source address, but the packet
     still leaves the tap toward its real destination; keying on `src == guest`
     would let the guest suppress its own egress from the summary by spoofing.
     Egress = an IPv4 TCP/UDP frame whose destination is not gateway-local (the
     host-side /30 peer or the guest itself); ARP/IPv6 and DHCP client/server
     ports are excluded as link-local noise. Return traffic (destined to the
     guest) is excluded by the non-local-destination rule, so direction never
     depends on the forgeable source.
   - `dns_queries`: best-effort list of queried names (from port-53 payloads),
     empty when none observed.
6. **Lifecycle events.** Emit `witness_started {tap}` and
   `witness_done {destinations, complete}` on the existing `--lifecycle`
   NDJSON stream, following the current event vocabulary (rooms-native: name
   what the substrate did).
7. **Failure posture.** Once capture has started, a mid-run capture failure
   must not kill the workload — the run continues and `witness.json` records
   `capture_complete: false`. Only the *initial* start (req. 3) fails closed.
8. **`rooms diff` / registry untouched.** No behavior change for runs without
   `--witness`.

## Non-functional requirements

- Follows the existing layering: capture process management belongs beside the
  other host-process mechanisms (`firecracker`/`transport` tier), summary
  parsing + schema in `artifacts`, wiring in `main`. No downward imports.
- Atomic writes for both artifacts (temp + rename), per repo convention.
- Unit-testable without KVM: pcap→summary parsing is pure and must be covered
  by unit tests over fixture pcaps (hand-built or checked-in tiny fixtures).
  Process lifecycle gets an e2e test behind the existing `e2e` feature.
- Errors not capitalized; no `unwrap`/`panic!` in non-test code; clippy strict
  per repo lints.

## Acceptance

- [ ] `rooms run --witness --command 'wget -q -O- http://93.184.216.34/ || true'`
      (writable image, e2e) produces `out/witness.json` listing that
      destination with `capture_complete: true`, and `witness.pcap` exists.
- [ ] The same run *without* `--witness` produces neither artifact and no new
      lifecycle events.
- [ ] Adversarial check (e2e): a guest command that contacts a destination and
      then tampers with its own `out/` (e.g. deletes guest-side traces) still
      yields that destination in `witness.json` — the witness does not depend
      on guest cooperation.
- [ ] `--witness` on a host without `tcpdump` fails before VMM start with a
      clear message.
- [ ] Unit tests: summary parser handles empty capture, truncated pcap, DNS
      extraction, gateway-noise exclusion.
- [ ] `make check` green; weighted LOC within the ideal band (< 700).

## Explicitly out of scope (v0)

- **Filesystem witness.** The overlay upper is a guest-side tmpfs
  (`overlay-init.sh`), so there is no host-visible upperdir to diff. A host
  witness for writes requires re-plumbing the upper onto a host-backed scratch
  disk — a real design change, deferred to its own spec.
- **Verdicts.** The witness emits evidence only. Deciding whether a
  destination set is acceptable belongs to the verification plane, not rooms.
- **eBPF.** tcpdump on the tap is sufficient for v0; an eBPF capture path is a
  later optimization, not a requirement.
- **In-guest syscall telemetry** — forgeable by design; permanently out of
  scope for the *witness* (that's what makes it a witness).
